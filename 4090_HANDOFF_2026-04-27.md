- 当前剩余阻塞：环境包冲突，不是流程问题。

## 2) 已经踩过并确认的坑

- `GatedRepoError 401`：`FLUX.1-dev` 需要 HF 登录与权限。
- `max_train_steps` 与 `num_train_epochs` 冲突：需设 `num_train_epochs=0`。
- `Setting accelerator to CPU`：用了错误 torch 组合导致 GPU 没真正起效。
- `deepspeed ... CUDA_HOME does not exist`：单卡 LoRA 不需要 deepspeed，直接卸载。
- `PIL.UnidentifiedImageError`：数据集中混入 Mac 资源叉文件（`._xxx.jpg`）。
- `torchao` 版本冲突：`peft` 要求 `torchao>0.16`，而现有是 `0.15.0`。

## 3) 新租 4090 后的最短可复现步骤（推荐）

以下命令在云端项目目录执行（假设目录为 `~/SimpleTuner`）：

```bash
cd ~/SimpleTuner

# 1) 建议先用 py3.12 环境
conda create -n st312 python=3.12 -y
conda activate st312

# 2) 安装项目（源码模式）
pip install -e '.[cuda]'

# 3) 避免已知冲突（单卡 LoRA）
pip uninstall -y deepspeed torchao

# 4) 登录 HF（用于 FLUX.1-dev）
hf auth login

# 5) 清理 Mac 垃圾文件与坏缓存（每次换机都建议执行一次）
find datasets/person_lora -type f -name '._*' -delete
find datasets/person_lora -type f ! \( -iname '*.jpg' -o -iname '*.jpeg' -o -iname '*.png' -o -iname '*.webp' \) -delete
rm -rf cache/vae/flux/person-lora

# 6) 核验 GPU
python -c "import torch; print('torch=',torch.__version__,'cuda=',torch.cuda.is_available(),'cuda_ver=',torch.version.cuda)"

# 7) 开训
export HF_HUB_DISABLE_XET=1
export HF_HUB_ENABLE_HF_TRANSFER=1
python -m simpletuner.cli train 2>&1 | tee train.log
```
